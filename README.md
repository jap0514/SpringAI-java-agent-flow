该系统执行的完整流程如下：
1、用户在前端提出问题

2、MCP接收到问题，并且设定好整个调用链路的最大执行次数，开始循环执行

3、先不着急直接用大模型来得出答案，而是先根据用户的问题来生成规范的、准确的一个执行计划（因为用户不可能像程序员一样具体的知道该怎么写提示词，所以所提的问题通常口语化，需要转换一下）
- 判断是否需要调用内部工具。如果需要用到工具的话，给出工具的具体名字或者相对应的映射
- 判断是否需要向量检索
- 判断是否需要用到历史对话
- 判断用户的问题是否需要进行进一步的理解。如果需要查询理解的话，那么理解、修改完的问题是什么。

4、得到一个准确的执行计划

5、执行计划
- 查询理解
- 工具调用
- 向量检索
- 模型调用

6、执行完之后需要进行反思reflection

- 判断得到的答案是否已经能够回答用户提的问题，如果可以就直接返回；
- 如果不完整那就继续循环执行

7、保存到历史会话，待下一次调用
